WEB ACCESSABILITY: FORMS

Forms are a commonly found within web applications and offer users the interaction to submit data to the site's backend server. For instance, forms are used for registering, logging in, commenting, purchasing and much more. Since they are used so oftenly, it is important to implement forms that are accessible for everyone. These concepts will apply for all online forms, no matter if they are processed client or server side. You can find multiple forms within our website. For example, check out our regististration form.

Form Layout

When creating an online form, there are many things to keep in mind to keep ADA compliance. Setting up a form correctly involves paying attention to your form layout and input design.

An example of the design process of a form can be seen to the right. It may seem enough for a form to highlight an incorrect field as red, but that is not enough to be ADA compliant. A form should inform the user of the error which can be done using JavaScript (that way, a screen reader will read out the error while reading over the label).

When there is more than one input element, a fieldset element should be used to group them if they are related. Each input element will need to inlcude a label or title attribute in conjunction with placeholder text for form inputs. Labels \ should be immediately placed to the left of the field as well. This is important because placeholder text is not enough for assistive technology to pick up on. Labels should be unique for each field and not used on more than one field. This is due to screen readers not supporting labels that have been associated to multiple form fields or fields that have more than one label associated to them.

Form Inputs

There are a wide range of different inputs a form can receive including text inputs, textarea elements, telephone numbers, address entry, select elements, checkboxes, radio buttons, and date inputs. Depending on the input type, there are some basic structure your form needs to be ADA compliant.

All form inputs should have a related, unique label that either wraps it, or precedes it. The label for attribute should match its input’s id element, regardless of whether it wraps the element or not.

If the input is required, put “Required” or “(required)” within a span in the label. Using just * to indicate required inputs will not often get picked up by screen readers.

  • For example: <label for="name">"First Name"<span>(required)</span></label> would be the correct format.
  • Within the input, using an HTML 5 required='' attribute will allow for JavaScript form validation. The screen reader will also be able to read this.
  • In required inputs, include a regular expression pattern information for screen readers to understand.
When it comes to using CSS with forms, the element order should not be arranged with CSS. Form controls should be displayed to the user in the same order as they are written with HTML. This relates to ADA compliancy because screen readers narrate forms in the order the appear in the HTML file.

Google Forms

There are many programs and resources available that will create forms for you. Our team reccomends Google forms which are ADA compliant. When setting up a Google form, it is important to provide an introduction and instructions in the provided text box to keep ADA compliancy.

Why is it important for ADA compliant forms?

People with disabilities benefit from accessible forms which can become visually and cognitively complex without proper structure. Following a ADA compliant form layout allows all users to enjoy it.

  • People with cognitive disabilities can better understand accessible forms since they have improved layout structure, instructions and feedback.
  • People with limited dexterity benefit from large clickable labels which can be created for checkboxes and radio buttons.
  • People using speech input benefit from using form labels via voice coomands to select a field or activate a control.
  • People who need screen readers can find and understand form controls due to proper labels, fieldsets and other elements.
